# Доработка графиков: monthly, facet, scatter и квадрант доходность/дисконт

Документ фиксирует новый этап точечной доработки визуализаций OFZ_ANALITICS поверх текущего состояния проекта.

## Общие ограничения

- Проект не переписывается с нуля.
- `data/raw/` не изменяется.
- Текущая структура `outputs/charts/` сохраняется.
- Новые HTML-графики должны сохраняться через тематические подпапки `outputs/charts/`.
- Команды запуска в документации должны использовать локальный Python из корня проекта:

```powershell
.\.venv\Scripts\python.exe
```

## Проблемы, выявленные ручной проверкой

1. На `monthly_demand_supply` и аналогичных grouped bar charts не хватает подписей данных.
2. На `monthly_cumulative_placement` и аналогичных line charts не хватает подписей данных.
3. На `monthly_placement_by_format` и аналогичных facet-графиках несколько раз отображается подпись оси Y.
4. На `demand_cutoff_explanation` не применены улучшения размера окружностей: размер точки не объяснен и визуально плохо отражает объем размещения.
5. Не хватает нового квадранта, где:
   - X = дисконт к номиналу;
   - Y = доходность.

## План доработок

### 1. Grouped bar charts

Цель: добавить читаемые подписи значений на `monthly_demand_supply` и аналогичные grouped bar charts.

Ожидаемый результат:

- значения подписаны в млрд рублей или в единицах соответствующей оси;
- подписи не перегружают график;
- hover сохраняет полную детализацию.

### 2. Line charts

Цель: добавить подписи на `monthly_cumulative_placement` и аналогичные line charts.

Ожидаемый результат:

- подписываются ключевые точки, а не каждая точка без необходимости;
- для накопленного объема используется подпись `Накопленный объем размещения по номиналу, млрд рублей`;
- значения отображаются без технических суффиксов `M`, `B`, `k`.

### 3. Facet charts

Цель: убрать дублирующиеся подписи оси Y на `monthly_placement_by_format` и аналогичных facet-графиках.

Ожидаемый результат:

- Y-axis title отображается один раз или визуально не дублируется в каждой панели;
- панели остаются читаемыми;
- единицы измерения сохраняются.

### 4. Scatter demand cutoff

Цель: улучшить визуальное кодирование размера точки на `demand_cutoff_explanation`.

Ожидаемый результат:

- если размер точки = `placement_volume`, это явно указано в подзаголовке или аннотации;
- размер окружностей визуально различим;
- при необходимости применяется фиксированный размер с сохранением объема в hover;
- chart data сохраняет поля, объясняющие size policy.

### 5. Новый квадрант доходность/дисконт

Цель: добавить новый scatter/quadrant chart:

- X = `discount_to_nominal`;
- Y = доходность (`cutoff_yield`, если доступна, иначе `weighted_avg_yield`);
- размер точки = объем размещения по номиналу, если читаемо;
- цвет/символ выбираются по аналитически полезному разрезу без перегрузки.

Ожидаемый результат:

- новый HTML-график сохраняется в тематическую подпапку `outputs/charts/scatter/` или `outputs/charts/risk/` в зависимости от финального назначения;
- chart data export сохраняется в актуальную подпапку `outputs/exports/chart_data/`;
- методология описана в `docs/04_visualization/visualization_strategy.md` и ограничениях графиков.

## Проверки готовности этапа

- `py_compile` измененных Python-файлов.
- `html_chart_qa.py` учитывает новые правила подписей и новый график.
- Новые графики не сохраняются в корень `outputs/charts/`.
- `quality_gate.py --fast` не ломается из-за новой структуры.
