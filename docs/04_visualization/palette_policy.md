# Цветовая политика визуализаций

Дата обновления: `2026-05-25`.

## Общие правила

- Цвета должны быть контрастными и различимыми на экране и в HTML-экспорте.
- В stacked structure charts нельзя использовать синий и желтый как соседние структурные цвета.
- Цвета категорий должны помогать чтению структуры, а не конкурировать с подписями данных.

## Сроковая структура

Для графиков `maturity_structure` и `monthly_placement_by_maturity` используется порядок и палитра:

| Сегмент | Цветовой смысл |
|---|---|
| Долгосрочные | темно-фиолетовый / пурпурный |
| Среднесрочные | teal / бирюзовый |
| Краткосрочные | коралловый / терракотовый |
| Требует проверки | серый |

## Stacked structure charts

- Внутренние подписи показываются только для достаточно крупных сегментов.
- Итоговая сумма выводится сверху над столбцом, если в нем есть два и более сегмента.
- Hover должен содержать значение сегмента, долю сегмента в столбце, итог столбца и долю сегмента в общей сумме.

## Реализация

Цветовые константы централизованы в `scripts/palette.py`:

- `QUALITATIVE_PALETTE`;
- `SEQUENTIAL_PALETTE`;
- `STATUS_PALETTE`;
- `MATURITY_COLOR_MAP`;
- `FORMAT_COLOR_MAP`;
- `STRUCTURE_PALETTE`;
- `MATURITY_CATEGORY_ORDER`.
