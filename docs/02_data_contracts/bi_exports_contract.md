# Контракт BI-экспортов

Дата: 2026-06-16.

BI package является external generated artifact. Он собирается из уже созданных pipeline outputs и не коммитится в Git.

## Путь пакета

```text
releases/bi/ofz_analytics_bi_<report_date>_<period_type>_<aggregation_mode>_r<N>_<timestamp>/
```

`releases/` игнорируется Git.

## Обязательный состав

| Component | Source | Package path | Required |
|---|---|---|---|
| Dashboard exports | `outputs/dashboards/**/*.csv`, scoped by report parameters | `source/outputs/dashboards/` | yes |
| Semantic model v2 | `outputs/dashboards/semantic_model_v2/*` | `source/outputs/dashboards/semantic_model_v2/` | yes |
| Analytical CSV | `outputs/exports/analytical_csv/*_<scope>.csv` | `source/outputs/exports/analytical_csv/` | yes |
| Monthly metrics CSV | `outputs/exports/analytical_csv/monthly_metrics_<scope>.csv` | `source/outputs/exports/analytical_csv/` | yes |
| Revenue analytics CSV | `outputs/exports/analytical_csv/revenue_*_<scope>.csv` | `source/outputs/exports/analytical_csv/` | yes |
| Chart data CSV | `outputs/exports/chart_data/**/*_<scope>.csv` | `source/outputs/exports/chart_data/` | yes |
| Data dictionary | semantic `field_dictionary.csv` and dashboard data dictionaries | `source/...` | yes |
| KPI dictionary | semantic `kpi_dictionary.csv` and KPI summary exports | `source/...` | yes |
| Period dimension | generated from `monthly_metrics_<scope>.csv` | `dimensions/period_dimension.csv` | yes |
| OFZ type dimension | generated from dashboard auction-level export | `dimensions/ofz_type_dimension.csv` | yes |
| Placement format dimension | generated from dashboard auction-level export | `dimensions/placement_format_dimension.csv` | yes |
| BI README | generated by package builder | `README.md` | yes |
| BI manifest | generated by package builder | `bi_manifest.json`, `bi_manifest.md` | yes |

`<scope>` means:

```text
_<period_type>_<aggregation_mode>_<report_date>_retrospective_<N>
```

## Политика безопасности

- Missing required datasets must not be silently replaced with empty files.
- Dry-run reports missing items but writes nothing.
- Build mode exits non-zero if any required item is missing.
- Build mode requires both:
  - `--include-outputs`;
  - `--confirm BUILD_BI_PACKAGE`.
- Generated BI package contents under `releases/bi/` must not be committed.

## Единицы измерения

BI consumers must keep project unit conventions:

- `*_volume_bln`: billion rubles, unit field value `млрд рублей` when present.
- demand/supply raw dashboard fields without `_bln`: million rubles unless source dictionary states otherwise.
- yield fields: percent annualized.
- discount fields: percentage points.
- ratio fields: decimal ratio or percent according to source dictionary/display field.

## Политика nullable-полей

Nulls are allowed when the source data does not support a metric:

- `revenue_volume` can be null for historical rows without revenue data;
- `discount_to_nominal` can be null when cutoff price is unavailable;
- demand ratios can be null for rows where denominator is zero/missing;
- DRPA observations can have limited demand/supply fields.

Nulls must be preserved. BI consumers should not replace missing values with zero unless the measure contract explicitly defines zero as a real value.

## Поля качества

BI package consumers should retain:

- `data_quality_flag`;
- `data_quality_display`, when available;
- `source_markers`, when available;
- `is_incomplete_period`;
- `incomplete_period_reason`.

## Требования к manifest

`bi_manifest.json` records:

- package name and version;
- Git commit, branch and dirty flag;
- report parameters;
- generated timestamp;
- Python version;
- CLI command;
- raw data hashes;
- included files;
- generated BI helper tables;
- SHA256 checksums;
- missing required items.

## Связь с другими контрактами

BI package uses and does not replace:

- `processed_data_contract.md`;
- `analytical_tables_contract.md`;
- `chart_data_contract.md`;
- `dashboard_exports_contract.md`;
- `semantic_model_v2.md`.
