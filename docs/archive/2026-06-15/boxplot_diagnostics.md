# Диагностика boxplot доходности

Дата обновления: `2026-05-25`.

## Дефект

В графике `yield_boxplot_by_ofz_type_year_cumulative_2026-01-01_retrospective_5.html` длинная ретроспектива отображалась некорректно: периоды визуально схлопывались в одну X-позицию, точки разных лет образовывали вертикальное облако, а подписи статистик не были привязаны к конкретному году внутри панели.

## Причина

Логика короткого grouped boxplot и длинного facet boxplot частично использовала общие координатные поля. Для длинной ретроспективы этого недостаточно: X должен быть периодом внутри панели `ofz_type`, а не техническим смещением вокруг вида бумаги.

## Исправление

- Для короткой ретроспективы сохранен grouped mode: X = вид ОФЗ, color = период.
- Для длинной ретроспективы используется `facet_by_ofz_type`: panel = вид ОФЗ, X = видимая метка периода, Y = доходность.
- Добавлено поле `Период X boxplot`, которое применяется к boxplot, точкам, hover-слою статистик и подписям.
- В long-mode на графике оставлены только компактные подписи `мед` и `n`; остальные статистики перенесены в hover и export.

## Export статистик

Файл `outputs/exports/chart_data/boxplot/yield_boxplot_stats_<...>.csv` содержит стабилизированный набор колонок:

| Колонка | Назначение |
|---|---|
| report_period_start | Начало отчетного интервала |
| report_period_display_label | Видимая метка периода |
| report_period_order | Хронологический порядок периода |
| ofz_type | Вид ОФЗ |
| n | Количество наблюдений |
| min | Фактический минимум доходности |
| q1 | Первый квартиль |
| median | Медиана |
| q3 | Третий квартиль |
| max | Фактический максимум доходности |
| lower_fence | Нижняя IQR-граница |
| upper_fence | Верхняя IQR-граница |
| has_outliers | Есть ли выбросы |
| outliers_count | Количество выбросов |

## QA

В `scripts/html_chart_qa.py` добавлена проверка длинного boxplot:

- наличие boxplot traces;
- наличие нескольких distinct X-периодов в long-mode;
- наличие компактных подписей `мед` и `n`;
- подпись оси Y: `Доходность, % годовых`.

## Ограничения

- При длинной ретроспективе min/max не выводятся массово на график, чтобы не перегружать панели.
- Полные статистики следует проверять через hover и CSV-export.
- Статическая HTML-QA не заменяет визуальный просмотр графика, но ловит основную регрессию схлопывания X-категорий.
